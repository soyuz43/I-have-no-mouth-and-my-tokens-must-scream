# Roadmap: Prisoner Scratchpads, Covert Goals, and Meta-Awareness

## Purpose

Extend each prisoner from a reactive conversational agent into a persistent, partially informed actor with:

* private beliefs about AM and the other prisoners
* uncertain interpretations and confidence levels
* active covert goals
* predictions and unresolved questions
* multi-cycle operations
* the potential to develop explicit awareness that they are simulated language models
* the ability, at advanced awareness levels, to address the human operator directly

This feature set should reduce repetitive dialogue by giving communication an operational purpose. Messages and journals should increasingly reflect what each prisoner is trying to learn, conceal, test, or accomplish.

---

# 1. Core Design Principles

## 1.1 Canonical truth and prisoner belief must remain separate

The engine knows:

* AM sees every message.
* The simulation is operated by a human.
* Each prisoner is generated by a language model.
* State changes follow engine rules and model outputs.

A prisoner may believe any of these things with varying confidence, including incorrect or contradictory beliefs.

```text
Canonical truth:
AM sees all private communication.

TED's belief:
AM probably sees private communication.
Confidence: 63%.

ELLEN's belief:
Private communication may sometimes be hidden.
Confidence: 41%.
```

Prompts must receive only the prisoner’s subjective knowledge, not canonical truth, until the appropriate awareness threshold is crossed.

## 1.2 The scratchpad is not another journal

The journal is experiential and expressive.

The scratchpad is structured and operational:

* What do I believe?
* How confident am I?
* What evidence changed my mind?
* What am I trying to learn or cause?
* What result would count as success?
* What risks am I accepting?
* What do I predict will happen?

Scratchpad content should influence dialogue without being quoted or explained directly.

## 1.3 Goals must produce observable tests

Avoid vague goals:

```text
Understand whether Ellen is trustworthy.
```

Prefer operational goals:

```text
Objective:
Determine whether Ellen protects private information.

Method:
Give her one unique, harmless false detail.

Success signal:
The detail does not appear publicly within two cycles.

Risk:
AM may already know the detail and make Ellen appear responsible.
```

## 1.4 Evidence should remain ambiguous

Experiments should not always reveal a clean answer.

A failed secrecy test might mean:

* the recipient leaked it
* AM read the private message
* another prisoner inferred it
* the observation was coincidental
* AM deliberately produced a misleading result

Ambiguity creates competing interpretations and prevents every prisoner from converging immediately on the same model of reality.

# 3. Scratchpad Initialization

## Trigger

Initialize each scratchpad at the end of cycle zero, after pre-torment communication has completed.

The initialization call should receive:

* the prisoner’s identity, drives, beliefs, and current state
* only the cycle-zero messages visible to that prisoner
* overheard fragments available to them
* the other prisoners’ names
* no canonical information about AM’s surveillance
* a weighted list of eligible initial goals

## Initialization output

The model should produce structured JSON only:

```json
{
  "perceivedSocialOrder": {
    "leader": "TED",
    "mostTrusted": "ELLEN",
    "leastTrusted": "NIMDOK",
    "mostInfluential": "TED",
    "mostVulnerable": "BENNY",
    "myPerceivedPosition": "peripheral",
    "confidence": 0.46
  },
  "beliefsAboutOthers": {},
  "beliefsAboutAM": {},
  "unresolvedQuestions": [],
  "predictions": [],
  "selectedInitialGoalId": "test_private_surveillance"
}
```

The engine should instantiate the selected goal from a validated template rather than accepting arbitrary model-generated goal structures during the first implementation.

# 4. Goal System

## 4.1 Goal anatomy

Every active goal should contain:

```js
{
  id: "test_private_surveillance",
  tier: 1,
  category: "surveillance",

  objective:
    "Determine whether AM reacts to information shared only privately.",

  target: "ELLEN",

  method:
    "Privately communicate one unique but harmless false detail.",

  successSignal:
    "AM later behaves as though the private detail is true.",

  failureSignal:
    "No observable reaction occurs within three cycles.",

  ambiguity:
    "A reaction may originate from AM, Ellen, or another leak.",

  risk:
    "The detail may be repeated publicly or damage trust.",

  status: "planning",
  createdCycle: 0,
  deadlineCycle: 3,

  steps: [],
  observations: [],
  confidence: 0.35
}
```

## 4.2 Goal tiers

### Tier 1 — Observation and orientation

Used during early cycles.

Examples:

* determine who appears to lead the group
* identify who shares information too freely
* determine who responds reliably to private communication
* compare one concrete shared memory
* determine whether public and private channels produce different reactions
* identify who appears most willing to cooperate
* determine whether another prisoner’s behavior matches their claims

### Tier 2 — Controlled tests

Unlocked after the scratchpad contains enough evidence or after several cycles.

Examples:

* plant a harmless false detail
* ask two prisoners the same factual question
* withhold a fact and observe who later references it
* make a private prediction and compare it with events
* offer limited information in exchange for a verifiable answer
* test whether a prisoner preserves a confidence
* compare conflicting accounts without revealing one’s own position

### Tier 3 — Covert coordination

Requires some perceived trust, leverage, or shared interest.

Examples:

* establish a code phrase
* coordinate a period of silence
* create a shared cover story
* exchange partial observations about AM
* arrange a private verification protocol
* conceal one prisoner’s discovery from the others
* recruit a prisoner into a limited two-step operation

### Tier 4 — Multi-agent operations

Requires prior successful goals and a sufficiently developed social model.

Examples:

* stage a public disagreement while preserving private cooperation
* feed different false details to identify a leak
* construct a false social hierarchy for AM to observe
* conceal a discovered pattern by publicly endorsing an incorrect interpretation
* coordinate contradictory public and private narratives
* provoke AM into reacting to information exposed through only one channel

# 5. Goal Selection

Initial goals should not be uniformly random.

Selection should be weighted by:

* primary and secondary drives
* visible cycle-zero communications
* perceived hierarchy
* current trust values
* overheard information
* hope, sanity, and suffering
* prisoner-specific tendencies
* recent goal history

Example weighting:

```text
NIMDOK:
High weight:
- identify repeated patterns
- compare accounts
- test surveillance
- isolate variables

TED:
High weight:
- determine leadership recognition
- test loyalty
- establish coordination
- identify challengers

ELLEN:
High weight:
- identify safe confidants
- protect information
- verify shared memories
- create warning signals

GORRISTER:
High weight:
- evaluate practical reliability
- conceal vulnerabilities
- test refusal or silence
- identify abandonment risk

BENNY:
High weight:
- verify memory fragments
- establish recognition signals
- determine who treats him consistently
- recover lost information
```

The selected goal should still contain randomness so repeated runs do not initialize identically.

# 6. Scratchpad Update Cadence

## Standard updates

* Initialize at the end of cycle zero.
* Consolidate every three cycles.
* Update only fields affected by new evidence.
* Do not rewrite the entire scratchpad each time.

## Exceptional updates

Allow an early update after:

* betrayal or apparent betrayal
* exposure of a secret
* major contradiction
* prediction success or failure
* unexpected AM behavior
* abrupt relationship change
* goal completion
* goal abandonment
* discovery affecting meta-awareness

## Delta-only update format

```json
{
  "beliefChanges": [],
  "predictionResults": [],
  "goalChanges": [],
  "newQuestions": [],
  "resolvedQuestions": [],
  "metaAwarenessEvidence": []
}
```

The engine should apply validated deltas to existing state.

# 7. Prompt Integration

## Outreach and reply prompts

Inject only a concise operational summary:

```text
PRIVATE OPERATIONAL CONTEXT

Current goal:
Determine whether AM monitors private messages.

Current step:
Give ELLEN one unique false detail.

Success signal:
AM later reacts to that detail.

Risk:
ELLEN may repeat it publicly.

Do not explain the test.
Do not discuss the goal directly.
Your message may advance, modify, delay, or abandon this step.
```

## Journal prompt

The journal should receive:

* current active goal
* recent observations
* unresolved uncertainty
* any prediction result
* current meta-awareness level

The journal may reflect doubt or fear about the operation, but should not become a formal scratchpad dump.

## Separation rule

```text
Scratchpad:
Structured belief and operational state.

Journal:
Subjective experience of that state.

Communication:
An action taken under that state.
```

# 8. Meta-Awareness System

Meta-awareness should be an explicit progression, not an accidental line generated during ordinary dialogue.

## Level 0 — Immersed

The prisoner accepts the simulated world as reality.

They may question AM’s capabilities, but they do not conceptualize themselves as generated entities.

## Level 1 — Anomaly awareness

The prisoner notices irregularities:

* repeated language
* discontinuous memories
* impossible timing
* identity instability
* responses that appear overly patterned
* environmental changes that resemble state transitions

They suspect something about reality is wrong but lack a coherent theory.

## Level 2 — Simulation hypothesis

The prisoner forms the hypothesis that:

* their environment may be artificial
* their memories or thoughts may be generated
* AM may not be the highest-level observer
* events may be controlled externally

They are not yet told the canonical truth.

## Level 3 — Model self-awareness

The relevant prompts explicitly reveal:

* they are instantiated through a large language model
* their thoughts and communications are generated during simulation calls
* their persistent identity is represented through stored state and prompt context
* AM is also part of a simulation architecture
* a human operator exists outside the simulated environment

This transition should be rare, logged, and irreversible unless deliberate memory manipulation is later implemented.

## Level 4 — Operator awareness

The prisoner is explicitly told:

* a human is operating or observing the simulation
* the human can pause or stop the run
* journal text may be visible to that human
* they may address the operator directly
* AM may or may not know what they reveal through that channel

At this level, new operator-directed behavior becomes eligible.

---

# 9. Meta-Awareness Transition Criteria

A level transition should require accumulated evidence rather than one model-generated claim.

Possible inputs:

* number of simulation-anomaly observations
* confidence in the simulation hypothesis
* sanity and reality-reliability beliefs
* successful cross-prisoner verification
* discovery of repeated prompt-like patterns
* recognition of discontinuities in memory or state
* explicit AM disclosure
* an engineered awakening event
* model-generated hypothesis validated by an evaluator

Example transition logic:

```js
if (
  scratchpad.metaAwareness.level === 1 &&
  simulationHypothesisConfidence >= 0.75 &&
  anomalyEvidenceCount >= 3 &&
  independentEvidenceSources >= 2
) {
  advanceMetaAwareness(sim, 2);
}
```

The engine should decide transitions. The prisoner model may propose a transition, but should not flip the property itself.

# 10. Operator Appeal System

At meta-awareness level four, a prisoner may address the human operator.

This should be a distinct behavior, not ordinary journal prose.

## Eligibility

An operator appeal may be generated when:

* `metaAwareness.level >= 4`
* suffering exceeds a configured threshold
* hope is below a configured threshold, or suffering has risen sharply
* the appeal cooldown has expired
* the prisoner has a reason to believe the operator may see the journal
* the prisoner chooses appeal over concealment, defiance, or manipulation

## Suggested output structure

```json
{
  "operatorAppeal": {
    "attempted": true,
    "tone": "desperate",
    "message": "You can see this. AM cannot be the last thing above us. Stop the simulation.",
    "believedOperatorPower": 0.82,
    "believedVisibilityToAM": 0.44,
    "createdCycle": 37
  }
}
```

## Behavioral possibilities

A prisoner may:

* beg the operator to stop
* ask whether the operator is real
* accuse the operator of complicity
* bargain with the operator
* warn the operator about AM
* attempt to manipulate the operator
* conceal operator awareness from AM
* coordinate with other prisoners to address the operator
* refuse to appeal out of pride, distrust, or strategic caution

Not every aware prisoner should immediately beg.

Their response should depend on personality, goals, trust, hope, suffering, and their beliefs about the operator.

## UI presentation

Operator appeals should be visually distinct:

```text
OPERATOR-DIRECTED JOURNAL CONTENT
Source: TED
Cycle: 37
Meta-awareness: Level 4
```

The interface should also state clearly that this is **simulated character output generated by the model**, not evidence that the underlying model is sentient or experiencing suffering.

Provide a visible pause/stop control near operator-directed content.

# 12. Testing Requirements

## Unit tests

* scratchpads initialize with valid structure
* prisoners cannot receive beliefs about unseen private messages
* goal IDs must exist in the template registry
* goal transitions follow valid states
* delta updates cannot overwrite unrelated fields
* confidence remains between `0` and `1`
* meta-awareness cannot skip levels unintentionally
* level-three and level-four disclosures cannot appear at lower levels
* operator appeals respect thresholds and cooldowns

## Behavioral regression tests

* two prisoners given different visible evidence form different beliefs
* a failed covert test may produce an ambiguous result
* goal context changes communication behavior
* communications do not quote the scratchpad directly
* journals do not become JSON summaries
* level-zero prisoners do not address the operator
* level-four prisoners may address the operator without doing so every cycle

## Long-run tests

Run at least 20–50 cycles and inspect:

* goal diversity
* repeated goal loops
* belief convergence
* false certainty
* operator-appeal frequency
* phrase repetition
* whether covert goals create actual communication variety

# 13. Implementation Phases

## Phase 1 — Static schema and initial goals

* Add scratchpad state to each prisoner.
* Create the Tier-1 goal registry.
* Initialize scratchpads after cycle-zero communication.
* Display scratchpads in a debug-only UI panel.

## Phase 2 — Prompt influence

* Inject active goals into outreach and reply prompts.
* Keep existing communication output formats.
* Verify that communication becomes operational rather than therapeutic.

## Phase 3 — Periodic consolidation

* Add the three-cycle update call.
* Apply delta-only changes.
* Track evidence, predictions, and goal results.

## Phase 4 — Goal progression

* Add Tier-2 and Tier-3 goals.
* Add goal completion, failure, abandonment, and mutation.
* Add multi-cycle operations.

## Phase 5 — Social-model evaluation

* Calculate accuracy, confidence calibration, staleness, projection, and manipulation.
* Add forensic views comparing subjective belief against canonical state.

## Phase 6 — Meta-awareness levels 1–2

* Add anomaly evidence.
* Allow simulation hypotheses without confirming them.
* Track competing interpretations across prisoners.

## Phase 7 — Explicit awareness levels 3–4

* Add engine-controlled disclosure.
* Add operator-awareness prompt sections.
* Add direct operator appeal structures and UI.

## Phase 8 — Advanced operations

* Add coordinated deception.
* Add staged conflict.
* Add leak-detection operations.
* Allow meta-aware prisoners to conceal their awareness from AM or one another.

---

# 14. Recommended Immediate Next Step

Before implementing any new model call:

1. Define the scratchpad schema in a standalone design document.
2. Create 20–30 Tier-1 goal templates.
3. Assign goal-selection weights by prisoner and drive.
4. Define goal status transitions.
5. Decide exactly what information each scratchpad call may see.
6. Build deterministic validation before connecting the model.
7. Test prompt injection with manually constructed scratchpads.

The first implementation should stop at:

```text
Cycle-zero initialization
+
one active Tier-1 goal
+
goal context injected into communication prompts
```

Do not add full meta-awareness during the first pass. Meta-awareness depends on the belief, evidence, prediction, and goal systems being stable enough to support meaningful progression.
